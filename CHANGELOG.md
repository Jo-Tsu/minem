# MineM 更新记录

本记录遵循 [Semantic Versioning](https://semver.org/lang/zh-CN/)；每次产品改动必须先更新 `product-version.json`、本文件和相应需求/技术文档。

## [0.4.0] - 2026-07-19

### 公开发行

- 首次依据 Apache-2.0 发布 MineM 源码，公开仓库使用不含内部 Git 历史的干净快照。
- 新增中英文 README、开源边界、贡献指南、安全策略、支持说明、Issue Forms 和 Pull Request 模板。
- 新增本机一键启动、默认空素材库的 Docker 配置、GitHub Actions、CodeQL、Dependabot 与发布边界检查。

### 变更

- macOS 快捷浮窗升级为 MineM 创作助手，仅保留汇报与页面双模式，不引入素材模式。
- 汇报模式支持导入、新建、插入、替换、修改和案例提炼；页面模式支持导入、新建、复制和修改。
- 支持识别客户端当前页面与剪贴板中的 `RPT-*`、`CTRL-PAGE-*` 和真实链接，通用 extracted 链接不再被猜测分类。
- 复制内容升级为 `minem.codex-operation.v1` 操作包，固定携带单页、多页归类、版本保留、16:9 和结果回传约束。
- 新增 `Option + Command + M` 全局唤起、客户端内快捷键、自定义指令模板与创作助手核心测试。

## [0.3.8] - 2026-07-19

### 变更

- 未声明汇报 Manifest 的单个 HTML 与独立多 HTML 包统一归入页面素材；只有明确多页 Deck 或汇报 Manifest 才生成汇报素材。
- 上传解析、素材写入与汇报/页面关系写入使用同一数据库事务，失败时整体回滚，不再留下半成品记录。
- ZIP 包按相对路径和实际文件字节生成标准化指纹，忽略压缩时间等元数据；相同内容改名或重新压缩后复用历史批次。
- 本机直接导入按文件内容去重，停止写入旧标签；服务启动默认不再自动扫描导入目录。
- 新增导入分类、标准化指纹和事务回滚回归测试，并纳入核心数据与点击测试流程。

## [0.3.7] - 2026-07-18

### 变更

- 工作台移除营销式 Hero 和重复统计；页面/资源列表使用紧凑吸顶工具条，低频资源操作收进“更多”。
- 案例卡片去掉重复说明和派生编号，优先使用页面缩略图，避免首屏加载多份完整案例汇报。
- 汇报与页面详情统一进入 `1920×1080` 平台画布，嵌入态隐藏查看器内部复制、刷新、全屏按钮。
- 尺寸适配包装页改用 HTTP 与本地 Chromium 均可解析的相对地址，并自动修复历史版本、刷新缩略图。
- 工作台与案例页隐藏无效搜索框，故事线详情合并重复字段并将低频结构信息折叠展示。
- 修复 `?arrange=<reportId>` 编排深链首次打开回退到汇报列表的问题。

## [0.3.6] - 2026-07-18

### 变更

- 删除前端设置与知识图谱页面、工作台入口、详情操作和图谱样式。
- 删除 `GET /api/graph`、`minem/graph.py`、图谱契约类型和内部 Agent 图谱任务规则。
- 原数据链路继续作为独立追溯能力保留；PRD、技术文档、设计规范和回归测试同步更新。

## [0.3.5] - 2026-07-18

### 修复

- 案例模块改为服务端读取已导入 manifest，并以真实 `CTRL-*` 主版本校验；删除前端 10 份静态案例 JSON 和内联案例常量。
- 移除 `server.py` 中 35 个零引用兼容函数和旧图谱构建器，拆出案例、工作台、导入任务前端组件。
- 修正文档中的旧数据量、旧脚本、旧代码行数和项目内数据库备份说明。
- 新增案例清单 API 契约与点击回归，确保案例组入口及关联页面可访问。

## [0.3.4] - 2026-07-18

### 修复

- Docker 素材源改为显式只读配置，平台与语音服务仅监听本机端口；内部 Agent API 改为显式启用。
- 导入批次使用运行目录相对路径，并为上传包记录内容指纹；重复上传直接复用历史素材。
- 清理 5 个零产物批次、23 份项目内历史数据库副本和旧 PPT 转 HTML 遗留。
- 版本脚本同步网页、桌面、Docker 与 `package-lock.json`，并补充运行目录回归检查。

## [0.3.3] - 2026-07-18

### 变更

- 素材详情版本侧栏与操作收纳优化

## [0.3.2] - 2026-07-15

### 修复

- 移除完整汇报 HTML 导出体积上限；大体积媒体保持原样并可随汇报导出。

## [0.3.1] - 2026-07-15

### 修复

- HTML 导出改为按页面静态依赖收集，排除 AI 演讲台、TTS、模型权重和语音运行时。
- 将 HTML 导出上限调整为 300MB；超限任务返回实际体积与明确原因。

## [0.3.0] - 2026-07-15

### 新增

- 完整汇报可按已确认编排导出离线 HTML 包或 PDF，隐藏和移出页面不会进入成品。
- HTML 离线包提供平台统一的翻页、复制链接、刷新和全屏能力；图片、视频、GIF 等媒体保持原始字节。
- 新增异步导出任务、进度查询、下载接口和导出回归检查。

## [0.2.0] - 2026-07-15

### 新增

- macOS AI 快捷浮窗：常用 MineM 操作指令复制、页面链接/编号校验和自定义模板。
- 统一预览工具栏：复制当前页链接、刷新和全屏。
- 汇报页面尺寸适配版本，保留原始页面素材与资源引用。

### 修复

- 修复导入单页被误归为汇报、预览路径失效、页面尺寸不一致和重复素材版本的问题。
- 统一多页汇报、独立页面、编排预览的翻页逻辑。

## 版本规则

- `MAJOR`：数据模型、公开 API 或导入语义存在不兼容变更。
- `MINOR`：新增用户可见能力，保持兼容。
- `PATCH`：修复、性能优化、文案或内部实现调整，保持兼容。
