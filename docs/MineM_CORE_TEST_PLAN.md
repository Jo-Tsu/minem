# MineM 核心功能测试用例与运行规范

版本：0.4.0 · 2026-07-19
状态：执行中
适用范围：MineM Web 平台、素材导入与治理、预览、汇报编排、故事线、AI 演讲台与 macOS 创作助手。

## 1. 目标与原则

本测试计划用于持续验证 MineM 的核心业务链路。测试主体只有两类：**数据正确性**和**用户点击可用性**。构建、类型和服务健康检查只是执行测试前的门槛，不作为主要验收结果。

重点防止以下已出现过的回归：

- 汇报、页面或资源素材的预览路径失效、返回 404、黑屏、空白或显示旧图。
- 同一次或失败重试后的导入重复创建汇报、页面、资源素材。
- 多页素材被错误识别为单页，或单页被错误识别为汇报。
- 页面编排、隐藏和插入后，预览页序与数据不一致。
- 资源合并、版本合并后出现资源丢失、空白图片或引用断裂。
- 打开链接、复制链接、刷新、翻页出现重复控件或失效。
- 平台构建、Docker 服务、API 契约与前端类型检查不一致。

测试脚本必须默认只读，不删除真实素材、不修改编排、不重新导入真实文件、不重建缩略图。涉及写入的验证统一使用临时目录、临时 SQLite 或专用测试数据。

## 2. 运行分层

### A. 数据正确性巡检

建议在每次功能改动、Docker 重建或发版前运行。目标时间：3 分钟以内。

1. 真实数据全量读取：每条活跃素材的 ID、编号、类型、来源路径、预览 URL、缩略图 URL、批次和版本关系完整。
2. 类型与编号：`RPT` 只能是汇报、`CTRL` 只能是单页页面、`RES` 只能是资源；案例产生的页面属于页面素材，不能被误判为汇报。
   - 这是阻塞级规则：任何 `CTRL-*`、`CTRL-PAGE-*` 或已识别为单页的来源文件出现在汇报列表，测试直接失败。
   - 导入汇报必须具备有效入口和至少一个可展示页面；导入后汇报预览、页面素材预览均不得黑屏、空白、乱码或只显示来源页背景。
3. 预览数据：每个素材的 `preview_url` 与真实文件一致；页面、汇报、资源不能出现 404、空路径、零字节或指向已删除文件。
4. 汇报页面关系：汇报的页面槽、编排顺序、隐藏状态、查看器输出和页面总数一致；不能包含其他汇报或案例的页面。
5. 资源引用：页面引用的图片、Logo、图标、GIF、视频均可解析；人工/系统合并后保留版本和引用，不能出现空白资源。
   - 递归验证 HTML、iframe、`data-fs-src`、`srcset`、CSS `url()` 与样式表依赖；来源包缺失本地资源时导入必须在入库前失败，不能生成“成功”任务或空白素材。
6. 重复数据：按内容哈希、预览入口、来源文件哈希、标题和页数识别重复；输出“重复”“同版本”“疑似相同但需人工确认”三类清单。
7. 缩略图正确性：缩略图来源与当前预览文件一致；素材更新后不能继续显示旧缩略图。
8. 导入数据：同一导入重试不会产生多条同内容汇报、页面或资源；外部多页文件会拆成页面素材，并按规则生成汇报或案例。
9. 故事线与临时预览：验证其数据隔离和引用完整，不得污染正式素材库。
10. 列表排序与分页：汇报、页面、资源均按最近业务活动时间倒序；默认每批固定 30 条，滚动加载不得改变排序。
11. 尺寸适配：公开汇报和嵌入页面必须使用统一的 `1920×1080` 等比画布；不得出现页面固定在左上、裁切、拉伸、黑边不对称或双重缩放。
12. 汇报尺寸版本化：页面尺寸与当前汇报目标画布不一致时，必须由汇报页槽引用一个可追溯的适配版本；原页面素材、资源和历史版本不得被覆盖或删除。
13. 历史适配页：当前页槽和历史版本中的尺寸包装页都必须使用可同时支持 HTTP 与本地截图的相对来源地址；禁止残留以 `/extracted/` 开头、在 `file://` 截图中失效的 iframe。

### B. 用户点击链路回归

建议在每周一次，或大批量导入、数据治理、合并逻辑变更后运行。目标时间：10 分钟以内，不生成预览图。

1. 列表卡片：点击汇报、页面、资源卡片都能打开对应详情，不打开错误素材。
2. 预览：点击详情、打开链接、复制链接、刷新、全屏均可用；打开链接必须在新标签中直接打开该素材当前 `preview_url`，不能错误跳转到平台查看器，也不依赖前一个窗口内存状态。
3. 汇报翻页：上一页、下一页、键盘方向键、页码状态可用；仅显示一套平台翻页与一套复制/刷新/全屏图标。
4. 页面编排：点击编排进入独立工作区；拖拽、隐藏/恢复、插入多页、确认编排、关闭时保存提示均可用。
5. 页面素材批量预览：勾选、多选、创建临时链接、翻页、关闭释放均可用。
6. 导入：打开导入、选择文件、提交、查看状态、关闭成功任务、失败提示和重试入口均可用。
7. 资源合并：选择候选、保留版本或生成新版本、查看合并后的资源预览和引用均可用。
8. 故事线：收藏、新建故事线、更新已有故事线版本、点击查看来源汇报均可用，且不新建汇报素材。
9. AI 演讲台：无稿时进入导入/生成；有稿时显示编辑/进入选项；讲稿编辑、分页和播放控制可用。
10. 列表交互：切换工作、汇报、页面、案例、资源、故事后回到顶部；页面/资源工具条吸顶且不遮挡内容；未选择时不展示无效批量按钮。
11. 客户可读性：常规桌面下汇报/页面卡片不被无限拉宽，资源卡片约 6 列且标题可辨识；案例卡片不重复解释字段或加载整份多页汇报。
12. macOS 创作助手：全局快捷键和菜单栏均可唤起；只显示汇报/页面双模式；当前页面与剪贴板识别、引用校验、自定义指令和复制操作包可用。

## 7. 不通过处理规则

1. `CORE-003` 至 `CORE-010` 失败：视为阻塞问题，停止发布，先修复链接、预览或查看器链路。
2. `CORE-012` 失败：不自动合并；输出相同内容的候选素材、来源批次和版本组，进入人工确认。
3. `DATA-001` 至 `DATA-005` 失败：生成数据修复计划，经人工确认后执行引用感知的修复。
4. 导入、编排、路径和运行目录治理用例失败：必须先在临时环境复现并补充回归用例。
5. 测试报告只记录素材 ID、路径和摘要，不写入素材正文、二进制资源或用户内容。

## 8. 脚本实施边界

脚本已按以下边界实现：

1. `smoke` 默认读取当前生产式本地数据、`audit` 的写入验证使用隔离临时库。
2. 脚本只报警，不自动删除重复素材、不自动合并版本、不自动重建缩略图。
3. 测试输出保存到 `artifacts/test-reports/`，该目录加入 `.gitignore`，不提交数据报告。
4. 导入幂等性、编排、故事线与临时预览用例采用临时 SQLite/目录测试，而非真实库写入。
