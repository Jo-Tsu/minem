# MineM macOS 客户端 PRD

> 状态：已确认并实施。包含“页面链接复制”“页面刷新”和“MineM 创作助手”。
>
> 本文记录当前已实施基线。下一版本“本地能力运行时与 AI 操作中心”的产品方案见 [MineM 本地 AI 客户端 PRD](MineM_LOCAL_AI_CLIENT_PRD.md)。

## 1. 背景

MineM 当前通过本地网页服务提供素材管理、导入、预览、故事线和 AI 演讲台。用户需要保留浏览器访问方式，同时获得可安装、可双击启动的 macOS 客户端，降低 Docker、Python 和本地地址的使用门槛。

## 2. 产品目标

1. 支持 macOS 用户安装 `MineM.app` 并双击启动。
2. 客户端与网页端使用同一套功能、界面和数据模型。
3. 客户端本机模式自动启动 MineM 服务，无需用户操作 Docker 或终端。
4. 保留浏览器访问能力，既支持本机地址，也支持未来团队部署地址。
5. 不改变现有素材生成、导入和 AI 能力的业务逻辑。
6. 让客户端能承载 MineM 内部链接，并提供清晰、受控的多标签导航体验。

## 3. 非目标

- 本期不开发 Windows 客户端。
- 本期不把用户素材上传到云端，也不改变本地数据所有权。
- 本期不替换现有 Python 后端、SQLite 或 React 前端。
- 本期不改变 Docker 的开发与部署用途。

## 4. 用户与场景

| 用户 | 场景 | 期望结果 |
| --- | --- | --- |
| 个人素材运营者 | 双击打开本地平台、导入和管理素材 | 无需先启动服务，素材仍保留在本机 |
| 汇报创作者 | 在客户端预览、创建故事线、使用 AI 演讲台 | 与网页端体验一致，预览正常可用 |
| 团队成员 | 使用浏览器或客户端访问团队服务 | 同一账号和权限下看到相同数据 |
| 开发与运维人员 | 使用 Docker 开发或部署 | 不与客户端数据目录和服务实例冲突 |

## 5. 核心流程

### 5.1 本机客户端首次启动

1. 用户从 `.dmg` 安装 `MineM.app`。
2. 用户双击应用。
3. 客户端创建专属本机数据目录。
4. 客户端启动本机 MineM 服务并进行健康检查。
5. 健康检查成功后，客户端进入素材库首页。
6. 后续再次打开时，保留所有本机素材、缩略图、故事线与设置。

### 5.2 浏览器访问

1. 用户通过浏览器访问本机或团队服务 URL。
2. 浏览器端调用既有 API。
3. 素材列表、导入任务、预览和详情与客户端保持一致。

### 5.3 服务异常

1. 客户端发现服务未成功启动或健康检查超时。
2. 客户端显示启动失败原因、日志位置与“重试”操作。
3. 不加载空白平台页，也不创建重复服务实例。

### 5.4 客户端内导航

1. 用户在详情页点击“打开链接”、预览页链接，或页面通过新窗口方式发起 MineM 内部跳转。
2. 客户端在顶部标签栏中新建对应标签页，不丢失当前素材库页面。
3. 新标签页支持关闭；关闭后回到原页面。
4. `127.0.0.1` 本机 MineM 地址与已配置的团队服务地址在客户端内打开。
5. 非 MineM 的外部网页由 macOS 默认浏览器打开，不加载进客户端。

### 5.5 平台升级后的客户端更新

1. 开发人员修改 React 前端、本机服务、模板、素材处理或运行依赖后，执行统一桌面构建命令。
2. 构建生成新版本 `.dmg`；用户将新版本拖入“应用程序”覆盖旧版。
3. 应用包更新只替换程序与内置服务，`Application Support/MineM` 中的用户数据不被删除。
4. 仅数据库或用户素材发生变化时，不需要重新打包客户端。

## 7. 数据与安全

- 本机模式数据保存于 `~/Library/Application Support/MineM/`，不写入 `.app` 包体。
- 本机服务只监听 `127.0.0.1`，不向局域网暴露素材内容。
- 安装包不包含用户上传素材、SQLite、缩略图、导入路径或语音模型。
- 客户端不得绕过现有导入安全校验、文件白名单、SQLite 锁和素材去重规则。

## 8. 关键体验约束

- 客户端只负责启动和承载平台，不改变页面布局、预览尺寸和素材生成机制。
- 本机服务只启动一个实例；发现已有健康实例时复用，而不是重复占用端口。
- 客户端升级不能删除用户数据。
- 导入与缩略图生成等耗时任务继续由后端执行，不能阻塞客户端主界面。
- 不能依赖网页端自行处理新窗口；客户端必须显式接管新窗口请求并给出导航结果。

## 9. 成功指标

- 新用户从双击应用到看到素材库首页的成功率。
- 本机客户端启动失败率与平均启动耗时。
- 客户端与浏览器端的 API 和预览一致性。
- 升级后本机数据保留成功率。

## 10. 待确认事项

1. 本期是否只支持本机模式，还是同时交付“团队服务 URL”切换。
2. 是否需要 Apple Developer 签名与公证，供非开发环境直接安装。
3. 客户端名称是否确定为 `MineM`，应用图标是否已有正式资源。
4. 本机服务端口是否固定为 `8790`，或改为由客户端自动选择可用端口。

## 11. 本期确认决策

- 本期交付本机模式，客户端自动启动本机 MineM 服务；团队服务 URL 切换保留为后续 P1 能力。
- 本机服务默认使用 `8790`，若端口被其它服务占用，客户端显示明确失败原因，不接管或停止其它实例。
- 应用名称使用 `MineM`；正式图标与 Apple 签名、公证在发布阶段补齐。

## 12. 本次变更待确认

1. 客户端新增简化标签栏：主页标签固定，内容页可多开；标签页仅提供关闭能力。
2. MineM 内部链接在客户端标签页打开；未受信任的外部地址交给系统浏览器。
3. “打开链接”按钮在客户端中会新开内容标签，而不是静默失败。
4. 平台代码改动后的发布方式统一为 `npm run desktop:build`，然后以新 `.dmg` 覆盖安装；用户数据目录保持不动。
5. 每个可独立打开的详情或预览页面右上角新增无文字的复制图标，点击复制当前 URL；成功后仅给出短暂状态反馈。
6. 与复制图标并列新增刷新图标，点击重新加载当前页面，用于查看修改后或重新生成后的最新内容；刷新不修改数据、不重复导入。

## 12.1 MineM 创作助手

### 目标

在 MineM.app 之外提供常驻但不打扰的桌面创作助手，帮助用户把“要做什么、操作哪个汇报、引用哪些页面、在哪一页生效”准确复制给 Codex 类型工具。助手只生成和复制结构化操作包，不直接绕过 MineM 的权限、导入、编排、版本或去重接口。

助手只包含“汇报”和“页面”两个模式，不设置“素材模式”。资源素材的检索、合并、标签和治理仍在 MineM 管理平台完成，不进入创作助手。

### 唤起与窗口行为

1. 用户可从 MineM 菜单栏图标或全局快捷键 `Option + Command + M` 唤起浮窗；快捷键注册失败时保留菜单栏入口并记录错误。
2. 浮窗为小型无边框窗口，默认置顶、可拖动、点击外部自动隐藏；主客户端继续正常使用。
3. 浮窗关闭或隐藏不会停止 MineM 本机服务，也不会改变主窗口标签页。
4. 浮窗只显示图标、短标题、当前上下文和“复制操作包”主操作；每项提供悬浮提示，避免长段说明占用空间。
5. 浮窗内快捷键为：`Command + 1` 切换汇报模式、`Command + 2` 切换页面模式、`Command + K` 聚焦动作区、`Command + Enter` 或 `Command + Shift + C` 复制操作包、`Esc` 隐藏。

### 汇报模式

| 操作 | 必要上下文 | 操作包约束 |
| --- | --- | --- |
| 导入 | 外部汇报文件路径或链接 | 多页整体只生成一个汇报素材，每一页分别生成页面素材；重复导入使用内容指纹幂等处理 |
| 新建 | 汇报主题；页面引用可选 | 提供页面引用时严格按引用顺序组装，不复制页面和资源 |
| 插入 | 汇报链接/编号、目标页、一个或多个页面引用 | 只新增当前汇报的页面槽位，不新建或删除页面素材 |
| 替换 | 汇报链接/编号、目标页、替换页面引用 | 保留被替换页面素材，只更新当前汇报的页面槽位 |
| 修改 | 汇报链接/编号、目标页、修改要求 | 基于目标页创建页面新版本，只让当前汇报引用新版本 |
| 案例 | 外部文档；目标汇报可选 | 提炼案例组和多张单页案例素材，需要时再插入目标汇报 |

### 页面模式

| 操作 | 必要上下文 | 操作包约束 |
| --- | --- | --- |
| 导入 | 外部单页文件或链接 | 只生成一个页面素材；检测到多页时停止并提示改用汇报导入 |
| 新建 | 页面主题和表达要求 | 从零创建一个 16:9 单页页面素材，返回新编号与真实链接 |
| 复制 | 来源页面链接或 `CTRL-PAGE-*` 编号 | 创建独立的新版本，不覆盖来源页面 |
| 修改 | 来源页面链接或编号、修改要求 | 创建页面新版本，保留旧版本及全部既有引用 |

### 页面引用规则

1. 插入、替换、复制页面和页面修改必须要求至少一个有效页面编号或 MineM 页面链接；未提供时复制按钮禁用。
2. 助手可按用户动作读取客户端最近访问的 MineM 页面或当前剪贴板，解析 `/pages/{controlId}/index.html`、`CTRL-PAGE-*`、`/reports/{reportId}/index.html` 和 `RPT-*`。
3. 通用 `/extracted/.../index.html` 不能仅凭 URL 猜测为汇报或页面，只作为来源链接带入，防止再次把页面素材误分到汇报列表。
4. 用户可粘贴、删除或多选页面引用，并选择目标页；操作包必须保留引用顺序。
5. 操作包使用稳定字段：`schema`、`mode`、`action`、`report_ref`、`target_page`、`page_refs`、`source_ref`、`requirements`，并附可读中文约束。
6. 助手不读取、上传或保存剪贴板历史；仅在用户点击“粘贴”时读取当前剪贴板，并在当前浮窗会话中保留引用。

### 自定义快捷指令

1. 用户可新建、命名、编辑、删除和复用自定义指令模板；模板仅保存在本机客户端浮窗存储中，不上传 MineM 服务。
2. 模板支持变量：`{{mode}}`、`{{action}}`、`{{page_refs}}`、`{{report_ref}}`、`{{target_page}}`、`{{source_ref}}`、`{{requirements}}`，复制时由当前内容替换；未填写的变量替换为空文本。
3. 自定义模板默认不执行 MineM 操作，只复制文本。用户如在模板中引用页面素材，仍可通过“读取剪贴板/添加引用”带入当前页面链接与编号。
4. 自定义模板名称不可为空；单个模板正文上限为 8,000 字符，最多保留 30 条，避免浮窗存储无限增长。

### 验收标准

1. 浮窗可在主客户端之外快速唤起、隐藏和再次唤起，不影响客户端服务、导入任务或预览加载。
2. 汇报和页面模式下全部核心操作都可生成包含稳定字段的 Codex 操作包；界面不存在第三个素材模式。
3. 需要页面引用的操作不能在缺失有效引用时复制；汇报编号不能进入页面引用列表，通用 extracted 链接不能被错误分类。
4. 操作包必须携带页面单页约束、多页导入归类、版本不覆盖、16:9 预览、真实链接和验证结果要求。
5. 用户复制后，内容在 Codex 类型工具中可读、可追溯，且保留页面链接和编号。
6. 助手不执行导入、删除、编排确认等写操作，不引入重复导入、数据污染或服务并发问题。
