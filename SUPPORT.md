# 支持 / Support

## 中文

- 可复现的软件缺陷：使用 GitHub 的“缺陷报告”Issue Form。
- 数据分类、导入或预览异常：使用“数据或预览问题”Issue Form。
- 新能力建议：使用“功能建议”Issue Form。
- 安全漏洞或疑似密钥泄露：不要创建公开 Issue，请按照
  [SECURITY.md](SECURITY.md) 私下报告。

提交问题前请搜索已有 Issue，并使用最新发布版本复现。请提供 MineM 版本、启动
方式、操作步骤和已经脱敏的日志。不得上传客户材料、数据库、导入包、凭据、模型
权重或包含私人路径的截图。

本项目由社区按可用时间维护，不承诺响应时限，也不通过公开 Issue 提供私有数据
恢复、客户环境排障或定制开发服务。

## English

- Reproducible software defects: use the Bug Report issue form.
- Data classification, import, or preview problems: use the Data or Preview
  Issue form.
- Product proposals: use the Feature Request issue form.
- Vulnerabilities or suspected secret exposure: do not open a public issue;
  follow [SECURITY.md](SECURITY.md) for private reporting.

Search existing issues and reproduce with the latest release before filing.
Include the MineM version, startup mode, steps, and sanitized logs. Never
upload customer materials, databases, import packages, credentials, model
weights, or screenshots containing private paths.

MineM is maintained by the community as time permits. There is no guaranteed
response time, and public issues are not a channel for private data recovery,
customer-environment troubleshooting, or custom development.
